# CRM Docs

โฟลเดอร์นี้ใช้เก็บเอกสารงานของ `apps/crm`

## When to write here

- งานที่ owner หลักคือ CRM
- งานที่แตะ CRM page, route, component, data flow, หรือ API ภายในของ CRM
- งาน cross-app ที่ CRM เป็น entrypoint หลัก

## Document template

ทุกไฟล์ควรมีหัวข้อขั้นต่ำ:

1. เป้าหมาย
2. สิ่งที่แก้
3. ผลกระทบ
4. วิธีทดสอบ
5. ข้อค้างหรือข้อควรระวัง

## Naming

ตั้งชื่อไฟล์ให้สื่อ task โดยตรง เช่น:

- `customer-search.md`
- `receipt-feature.md`
- `checkout-discount-rules.md`
