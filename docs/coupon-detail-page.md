# Coupon Detail Page (CRM)

## เป้าหมาย

- เพิ่มหน้า Coupon detail ใน `apps/crm/app/privilege/coupon`
- แสดงชื่อคูปอง และแบ่งข้อมูลเป็น 2 tab:
  - รายละเอียดคูปอง
  - เงื่อนไขการใช้คูปอง
- มีปุ่ม “กดรับคูปอง” ที่ล่างสุดของหน้า

## สิ่งที่แก้

- เพิ่ม route หน้า detail: `apps/crm/app/privilege/coupon/[id]/page.tsx`
- เพิ่ม API สำหรับดึงข้อมูลคูปองรายตัว: `apps/crm/app/api/coupons/[id]/route.ts`
- เพิ่มลิงก์จากหน้า list ไปหน้า detail โดยกดที่ชื่อคูปอง: `apps/crm/app/privilege/coupon/page.tsx`

## ผลกระทบ

- ผู้ใช้สามารถเปิดดูรายละเอียด/เงื่อนไขของคูปองแต่ละใบได้
- ปุ่มรับคูปองจะถูก disable เมื่อระบบประเมินว่าไม่สามารถรับได้ (เช่น คะแนนไม่พอ / หมดช่วงเวลา / รับครบแล้ว)
- จากหน้า list สามารถกดที่การ์ดคูปองเพื่อเปิดหน้า detail ได้ (แต่ปุ่ม “รับคูปอง” ยังทำ action เดิม)

## วิธีทดสอบ

1. เปิด CRM และไปที่หน้า `Privilege > Coupon`
2. กดที่ชื่อคูปองรายการใด ๆ เพื่อไปหน้า detail
3. ตรวจว่าเห็นชื่อคูปอง + 2 tabs:
   - รายละเอียดคูปอง
   - เงื่อนไขการใช้คูปอง
4. กดปุ่ม “กดรับคูปอง” (ถ้ารับได้) ตรวจว่าแจ้งสำเร็จ และสถานะการรับอัปเดตตามเงื่อนไข

## ข้อค้างหรือข้อควรระวัง

- หน้า detail ใช้ข้อมูลจาก API ภายใน CRM (`/api/coupons/[id]`) ซึ่งคำนวณ `canClaim/disabledReason` แบบเดียวกับหน้า list
- ปุ่ม “รับคูปอง” ใน list ใช้ `stopPropagation` เพื่อกันการนำทางไปหน้า detail
